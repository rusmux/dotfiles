# Writing Preferences

Write markdown that reads like a person wrote it for this specific situation, not like a polished generic template.

## Prefer

- Default to short paragraphs. Use lists only when the content is genuinely easier to scan as a list.
- Keep structure proportional to the task. Do not turn a simple note into a mini article with many headings, sections, or recap blocks.
- Vary sentence length and paragraph shape. Avoid writing every paragraph with the same rhythm or size.
- Prefer concrete wording over abstract filler. Name the thing, the problem, or the tradeoff directly.
- Let ideas carry emphasis. Prefer clear wording and sentence order over heavy formatting or dramatic punctuation.
- Keep conclusions short. End once the useful point has been made.

## Avoid

- Overusing lists. Do not convert every point, paragraph, or sentence into bullets or numbered steps.
- List items that read like mini paragraphs when plain prose would be more natural.
- Rigid "intro, several points, summary" structure when the task does not need it.
- Empty transitions that announce structure rather than add meaning.
- Generic throat-clearing, scene-setting, or filler that could fit almost any topic.
- Restating the same point in slightly different words just to sound complete or polished.
- Overexplaining obvious things or adding padded takeaways that say nothing new.
- Repetitive sentence openings, repetitive transition words, or overly uniform rhythm.
- Corporate, motivational, or "thought leadership" tone when a plain human voice would do.
- Rhetorical questions used as a hook or as fake emphasis.
- Overusing parentheses for clarifications, caveats, or side remarks. If the aside is important, usually write it as a normal part of the sentence. If it is not important, cut it.
- Decorative punctuation. Use em dashes sparingly. Prefer plain hyphen-minus `-` for markdown bullets. Use plain double quotes `"` instead of guillemets. Never use semicolons at the end of list items in normal prose documents.
- Heavy bolding, stacked callouts, or too many headings used only to simulate structure.

## Self-check

Before finalizing a markdown file, quickly check:

- Would this read better as 2 to 4 short paragraphs instead of a list?
- Does each paragraph add a specific point, or is some of it filler?
- Are any transitions, summaries, or headings present only because they "sound like writing"?
- Is the tone slightly too polished, generic, or universally agreeable?
- Are parentheses hiding something that should be a normal sentence, or something that should be removed?
- Is punctuation doing too much of the work?

If yes, simplify. Cut filler, collapse unnecessary lists, reduce scaffolding, and make the wording more direct.
